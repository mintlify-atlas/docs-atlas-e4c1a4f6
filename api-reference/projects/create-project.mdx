---
title: 'Create Project'
openapi: 'POST /v1beta1/organizations/{org_id}/projects'
---

Create a new project within an organization. The authenticated user automatically becomes the owner of the newly created project.

## Path Parameters

<ParamField path="org_id" type="string" required>
  Organization ID where the project will be created. Must be a valid UUID.
</ParamField>

## Request Body

<ParamField body="body" type="object" required>
  Project details to create.
  
  <ParamField body="name" type="string" required>
    Unique name/slug for the project within the organization. Must be lowercase, alphanumeric with hyphens. Used in URLs and API calls.
    
    **Example**: `data-platform`, `mobile-app`
  </ParamField>
  
  <ParamField body="title" type="string" required>
    Human-readable display name for the project. Can contain spaces and special characters.
    
    **Example**: `Data Platform`, `Mobile Application`
  </ParamField>
  
  <ParamField body="org_id" type="string" required>
    Organization ID where the project will be created. Should match the path parameter.
  </ParamField>
  
  <ParamField body="metadata" type="object">
    Custom metadata for the project. Can contain any valid JSON structure. Must comply with the project metadata schema if validation is enabled.
    
    **Example**: `{"team": "engineering", "department": "data", "cost_center": "cc-1234"}`
  </ParamField>
</ParamField>

## Response

<ResponseField name="project" type="object">
  The newly created project object.
  
  <ResponseField name="id" type="string">
    Unique identifier for the project (UUID format).
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Unique name/slug for the project.
  </ResponseField>
  
  <ResponseField name="title" type="string">
    Human-readable display name for the project.
  </ResponseField>
  
  <ResponseField name="org_id" type="string">
    ID of the organization this project belongs to.
  </ResponseField>
  
  <ResponseField name="metadata" type="object">
    Custom metadata associated with the project.
  </ResponseField>
  
  <ResponseField name="created_at" type="timestamp">
    Timestamp when the project was created (RFC3339 format).
  </ResponseField>
  
  <ResponseField name="updated_at" type="timestamp">
    Timestamp when the project was last updated (RFC3339 format).
  </ResponseField>
  
  <ResponseField name="members_count" type="integer">
    Number of members in the project. Will be 1 for newly created projects (the creator).
  </ResponseField>
</ResponseField>

```go Go
package main

import (
    "context"
    "fmt"
    frontierv1beta1 "github.com/raystack/frontier/proto/v1beta1"
    "google.golang.org/protobuf/types/known/structpb"
)

func createProject(client frontierv1beta1.FrontierServiceClient) {
    metadata, _ := structpb.NewStruct(map[string]interface{}{
        "team":       "engineering",
        "department": "data",
    })
    
    req := &frontierv1beta1.CreateProjectRequest{
        Body: &frontierv1beta1.ProjectRequestBody{
            Name:     "data-platform",
            Title:    "Data Platform",
            OrgId:    "9f256f86-d3c4-4a6e-b89a-1e9a23c5d78e",
            Metadata: metadata,
        },
    }
    
    resp, err := client.CreateProject(context.Background(), req)
    if err != nil {
        panic(err)
    }
    
    fmt.Printf("Created project: %s (%s)\n", resp.Project.Title, resp.Project.Id)
}
```

## Error Responses

<ResponseField name="code" type="string">
  Error code indicating the type of error.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message.
</ResponseField>

### Common Errors

- **400 Bad Request**: Invalid request body or missing required fields
  - Invalid project name format (must be lowercase alphanumeric with hyphens)
  - Invalid organization ID format
  - Metadata doesn't match the required schema
  - Missing required fields (name, title, or org_id)
- **401 Unauthenticated**: Invalid or missing authentication token
- **404 Not Found**: Organization not found
- **409 Conflict**: Project with the same name already exists in the organization
- **500 Internal Server Error**: Server encountered an unexpected error

## Notes

- The authenticated user automatically becomes the project owner with full permissions
- Project names must be unique within an organization (but can be duplicated across different organizations)
- The project is created in `enabled` state by default
- An audit log entry is created for the project creation event
- Default owner role and permissions are automatically set up for the creator
- A relation is automatically created linking the project to its organization
