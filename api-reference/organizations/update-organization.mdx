---
title: 'Update Organization'
openapi: 'PUT /v1beta1/organizations/{id}'
---

Update an existing organization's details. Requires admin/owner permissions on the organization.

## Path Parameters

<ParamField path="id" type="string" required>
  The organization identifier. Can be either:
  - Organization UUID (e.g., `9f256f86-d3c4-4a6e-b89a-1e9a23c5d78e`)
  - Organization name/slug (e.g., `acme-corp`)
</ParamField>

## Request Body

<ParamField body="body" type="object" required>
  Organization fields to update. Only provided fields will be updated.
  
  <ParamField body="name" type="string">
    Unique name/slug for the organization. Must be lowercase, alphanumeric with hyphens.
    
    **Warning**: Changing the name affects all URLs and API references.
    
    **Example**: `acme-corporation`, `new-org-name`
  </ParamField>
  
  <ParamField body="title" type="string">
    Human-readable display name for the organization.
    
    **Example**: `Acme Corporation`, `New Organization Name`
  </ParamField>
  
  <ParamField body="avatar" type="string">
    URL to the organization's avatar image. Set to empty string to remove.
    
    **Example**: `https://example.com/avatars/new-logo.png`
  </ParamField>
  
  <ParamField body="metadata" type="object">
    Custom metadata for the organization. This completely replaces existing metadata - use a partial update pattern in your application if you want to preserve existing fields.
    
    **Example**: `{"industry": "fintech", "employees": 500, "region": "eu-west"}`
  </ParamField>
</ParamField>

## Response

<ResponseField name="organization" type="object">
  The updated organization object.
  
  <ResponseField name="id" type="string">
    Unique identifier for the organization (UUID format).
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Updated unique name/slug for the organization.
  </ResponseField>
  
  <ResponseField name="title" type="string">
    Updated human-readable display name.
  </ResponseField>
  
  <ResponseField name="avatar" type="string">
    Updated avatar URL.
  </ResponseField>
  
  <ResponseField name="state" type="string">
    Current state of the organization (unchanged by this endpoint).
  </ResponseField>
  
  <ResponseField name="metadata" type="object">
    Updated custom metadata.
  </ResponseField>
  
  <ResponseField name="created_at" type="timestamp">
    Original creation timestamp (unchanged).
  </ResponseField>
  
  <ResponseField name="updated_at" type="timestamp">
    Timestamp of this update (RFC3339 format).
  </ResponseField>
</ResponseField>

```go Go
package main

import (
    "context"
    "fmt"
    frontierv1beta1 "github.com/raystack/frontier/proto/v1beta1"
    "google.golang.org/protobuf/types/known/structpb"
)

func updateOrganization(client frontierv1beta1.FrontierServiceClient, orgID string) {
    metadata, _ := structpb.NewStruct(map[string]interface{}{
        "industry":  "technology",
        "size":      "enterprise",
        "employees": 5000,
    })
    
    req := &frontierv1beta1.UpdateOrganizationRequest{
        Id: orgID,
        Body: &frontierv1beta1.OrganizationRequestBody{
            Title:    "Acme Corporation Inc.",
            Avatar:   "https://example.com/avatars/acme-new.png",
            Metadata: metadata,
        },
    }
    
    resp, err := client.UpdateOrganization(context.Background(), req)
    if err != nil {
        panic(err)
    }
    
    fmt.Printf("Updated organization: %s\n", resp.Organization.Title)
    fmt.Printf("  Updated at: %s\n", resp.Organization.UpdatedAt)
}
```

## Error Responses

<ResponseField name="code" type="string">
  Error code indicating the type of error.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message.
</ResponseField>

### Common Errors

- **400 Bad Request**: Invalid request body or field values
  - Invalid organization name format
  - Metadata doesn't match the required schema
  - Empty request body
- **401 Unauthorized**: Invalid or missing authentication token
- **404 Not Found**: Organization doesn't exist
- **409 Conflict**: New organization name already exists (if changing name)
- **500 Internal Server Error**: Server encountered an unexpected error

## Notes

- Only organization admins/owners can update organization details
- You can update individual fields without providing all fields
- Changing the organization name affects all URLs and API references
- Metadata is replaced entirely, not merged - include all metadata fields you want to keep
- An audit log entry is created for the organization update event
- The `updated_at` timestamp is automatically set to the current time
- To remove the avatar, set it to an empty string
- Organization state cannot be changed through this endpoint - use enable/disable endpoints instead
