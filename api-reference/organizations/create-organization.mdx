---
title: 'Create Organization'
openapi: 'POST /v1beta1/organizations'
---

Create a new organization. The authenticated user automatically becomes the owner/admin of the newly created organization.

## Request Body

<ParamField body="body" type="object" required>
  Organization details to create.
  
  <ParamField body="name" type="string" required>
    Unique name/slug for the organization. Must be lowercase, alphanumeric with hyphens. Used in URLs and API calls. Cannot be changed after creation.
    
    **Example**: `acme-corp`, `my-organization`
  </ParamField>
  
  <ParamField body="title" type="string" required>
    Human-readable display name for the organization. Can contain spaces and special characters.
    
    **Example**: `Acme Corporation`, `My Organization`
  </ParamField>
  
  <ParamField body="avatar" type="string">
    URL to the organization's avatar image. Should be a publicly accessible image URL.
    
    **Example**: `https://example.com/avatars/acme.png`
  </ParamField>
  
  <ParamField body="metadata" type="object">
    Custom metadata for the organization. Can contain any valid JSON structure. Must comply with the organization metadata schema if validation is enabled.
    
    **Example**: `{"industry": "technology", "size": "enterprise", "region": "us-west"}`
  </ParamField>
</ParamField>

## Response

<ResponseField name="organization" type="object">
  The newly created organization object.
  
  <ResponseField name="id" type="string">
    Unique identifier for the organization (UUID format).
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Unique name/slug for the organization.
  </ResponseField>
  
  <ResponseField name="title" type="string">
    Human-readable display name for the organization.
  </ResponseField>
  
  <ResponseField name="avatar" type="string">
    URL to the organization's avatar image.
  </ResponseField>
  
  <ResponseField name="state" type="string">
    State of the organization. Newly created organizations are `enabled` by default.
  </ResponseField>
  
  <ResponseField name="metadata" type="object">
    Custom metadata associated with the organization.
  </ResponseField>
  
  <ResponseField name="created_at" type="timestamp">
    Timestamp when the organization was created (RFC3339 format).
  </ResponseField>
  
  <ResponseField name="updated_at" type="timestamp">
    Timestamp when the organization was last updated (RFC3339 format).
  </ResponseField>
</ResponseField>

```go Go
package main

import (
    "context"
    "fmt"
    frontierv1beta1 "github.com/raystack/frontier/proto/v1beta1"
    "google.golang.org/protobuf/types/known/structpb"
)

func createOrganization(client frontierv1beta1.FrontierServiceClient) {
    metadata, _ := structpb.NewStruct(map[string]interface{}{
        "industry": "technology",
        "size":     "enterprise",
    })
    
    req := &frontierv1beta1.CreateOrganizationRequest{
        Body: &frontierv1beta1.OrganizationRequestBody{
            Name:     "acme-corp",
            Title:    "Acme Corporation",
            Avatar:   "https://example.com/avatars/acme.png",
            Metadata: metadata,
        },
    }
    
    resp, err := client.CreateOrganization(context.Background(), req)
    if err != nil {
        panic(err)
    }
    
    fmt.Printf("Created organization: %s (%s)\n", resp.Organization.Title, resp.Organization.Id)
}
```

## Error Responses

<ResponseField name="code" type="string">
  Error code indicating the type of error.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message.
</ResponseField>

### Common Errors

- **400 Bad Request**: Invalid request body or missing required fields
  - Invalid organization name format (must be lowercase alphanumeric with hyphens)
  - Metadata doesn't match the required schema
  - Missing required fields (name or title)
- **401 Unauthenticated**: Invalid or missing authentication token
- **409 Conflict**: Organization with the same name already exists
- **500 Internal Server Error**: Server encountered an unexpected error

## Notes

- The authenticated user automatically becomes the organization owner
- Organization names are globally unique and cannot be changed
- The organization is created in `enabled` state by default
- An audit log entry is created for the organization creation event
- Default roles and permissions are automatically set up for the owner
